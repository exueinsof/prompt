Architettura e Stack Tecnologico (100% Open Source)
L'intera architettura sarà basata su microservizi containerizzati. È fondamentale rispettare il seguente stack tecnologico e, soprattutto, il principio di infrastruttura condivisa:
* Infrastruttura Comune (Condivisa):
   * Database Principale: Un'unica istanza PostgreSQL.
   * Estensioni DB: TimescaleDB (per tutti i dati time-series) e PostGIS (per i dati geospaziali).
   * Object Storage: Un'unica istanza Minio (che fungerà da Data Lake ).
   * Constraint Fondamentale: Tutti i servizi (Keycloak, ThingsBoard, Flowable, ecc.) devono essere configurati per utilizzare questa singola istanza PostgreSQL e Minio come backend di persistenza, per garantire l'unificazione dei dati.

   Le componenti sono le seguenti:
   1. Identità, Sicurezza & Audit (Compliance GDPR)
      Keycloak: SSO, RBAC e gestione delle identità.
      Grafana Loki: (Nuovo) Fondamentale per il GDPR. Centralizza tutti i log di sistema e degli accessi, permettendo di fare "forensics" e audit su chi ha fatto cosa e quando.
   2. Comunicazione & Nervo Centrale (Real-time)
      MQTT (Mosquitto): Il "sistema nervoso", MQTT permette il disaccoppiamento: i sensori pubblicano, i servizi (ThingsBoard, Netbox, Zabbix, HomeAssistant, NODE-RED, FastAPI, etc) si iscrivono.
   3. Acquisizione Dati & Ingestione (Ingestion)
      Apache NiFi: Per flussi dati complessi, trasformazioni ETL pesanti e integrazione file/DB.
      Node-RED: Per logica "edge" rapida, protocolli industriali leggeri e automazioni veloci.
      Home Assistant: Building Atuomation
      Zabbix: Per il monitoraggio dello stato di salute degli asset IT (server, switch, connettività).
   4. Persistenza Dati (Data Lakehouse)
      Minio: Data Lake per oggetti (immagini, documenti degli asset, dump massivi).
      Postgres + TimescaleDB + PostGIS: Il cuore dei dati strutturati (Metadati + Serie Temporali + Dati Geografici).
   5. Query & Federazione
      Trino: Il layer che permette di interrogare con un unico SQL sia i file su Minio che le tabelle su Postgres.
   6. Processi & Asset Management
       Flowable: Motore BPMN 2.0 per workflow
       ThingsBoard Community Edition: Gestione pura dei device IoT, dashboard di controllo e telemetria.
       Netbox: Fondamentale per il "Digital Twin" dell'infrastruttura fisica (cablaggi, rack, posizionamento negli armadi).
   7. Data Science, AI & Sviluppo Custom
      FastAPI (Python): Per creare microservizi e integrare modelli LLM/ML (Scikit-learn, Transformers).
      React + React Flow: Per l'interfaccia utente e la visualizzazione di grafi o schemi logici.
   8. BI & Digital Twin (Visualization)
      Apache Superset: Analisi dati e dashboarding BI di alto livello.
      Cesium.js & Three.js: Visualizzazione geografica (GIS) e modelli 3D (BIM/CAD) degli asset.

linguaggi preferiti: Python, React, JavaScript, HTML, Rust

main_thingsboard_admin.py (Logica Principale)
Genera il codice Python/FastAPI che implementi:
* Setup: Inizializzazione di FastAPI, CORS, caricamento .env.
* Sicurezza (Dependency): Una funzione get_current_admin_user che valida un token Keycloak (dall'header Authorization) e verifica che abbia il ruolo 'holos-admin'.
* Helper: Funzioni asincrone per ottenere e cachare i token admin di Keycloak e ThingsBoard (System Admin).
* Modelli Pydantic: CreateTenantRequest, CreateAssetRequest, CreateUserRequest, AssignUserToAssetRequest.
* Endpoint API per React Flow (protetti da get_current_admin_user):
   * GET /api/admin/graph-data/{tenant_id}: Recupera Tenant, Asset (Customer) e Utenti da ThingsBoard e li formatta per React Flow ({ "nodes": [...], "edges": [...] }).
   * POST /api/admin/tenant: Riceve CreateTenantRequest, crea il Tenant in ThingsBoard, restituisce il nodo Tenant per React Flow.
   * POST /api/admin/asset: Riceve CreateAssetRequest, crea il Customer (Asset) in ThingsBoard, restituisce il nodo Asset.
   * POST /api/admin/user: Riceve CreateUserRequest, crea l'utente sia in Keycloak (per il login) sia in ThingsBoard (come "ombra"), restituisce il nodo User.
   * POST /api/admin/assign-user-to-asset: Riceve AssignUserToAssetRequest, chiama l'API di ThingsBoard POST /api/customer/{assetId}/user/{userId}, restituisce il nuovo arco (edge).
* Endpoint API per Utente Finale:
   * GET /api/auth/thingboard-login (protetto da un get_current_user che valida qualsiasi utente).
   * Logica: Estrae l'email dal token Keycloak; trova l'ID utente "ombra" in ThingsBoard; trova la dashboard dell'asset a cui l'utente è assegnato; genera un token temporaneo (POST /api/user/{userId}/token); restituisce { "thingboardToken": "...", "dashboardUrl": "http://.../dashboard/ID_DASHBOARD?token=" }.

L'obiettivo è lo sviluppo end-to-end di HOLOS (Hub Olistico di Orchestrazione di Sistemi), un'architettura funzionale per la trasformazione digitale olistica. * Essere un'architettura 100% open-source, scalabile e multi-tenant.
* Implementare un'interfaccia di portale unica (Punto Unico di Accesso) per tutti gli utenti, basata su un rigoroso controllo degli accessi (RBAC).

1. Punto Unico di Accesso: Un portale React (holos.giuseppeceleste.it) funge da interfaccia per tutti gli utenti e tutti i moduli (Amministrazione, IoT, Processi, BI, Digital Twin, Data Science).

def load_json_file(path, default=None):
    """Carica un file JSON in modo sicuro."""
    try:
        with open(path, 'r') as f:
            return json.load(f)
    except (FileNotFoundError, json.JSONDecodeError):
        return default if default is not None else {}


def save_json_file(path, data):
    """Salva i dati in un file JSON."""
    os.makedirs(os.path.dirname(path), exist_ok=True)
    with open(path, 'w') as f:
        json.dump(data, f, indent=2)


def get_tb_jwt(session):
    """Ottiene un token JWT da ThingsBoard."""
    try:
        response = session.post(
            f"{TB_API_URL}/api/auth/login",
            json={"username": TB_USER, "password": TB_PASSWORD}
        )
        response.raise_for_status()
        return response.json()['token']
    except requests.exceptions.RequestException as e:
        logging.error(f"Errore durante il login a ThingsBoard: {e}")
        return None


def provision_devices(allowed_entities, device_map):
    """Crea i dispositivi su ThingsBoard se non esistono."""
    logging.info("Avvio provisioning dei dispositivi su ThingsBoard...")
    
    with requests.Session() as session:
        jwt_token = get_tb_jwt(session)
        if not jwt_token:
            return device_map


        session.headers.update({'X-Authorization': f'Bearer {jwt_token}'})
        
        updated = False
        for entity_id in allowed_entities:
            if entity_id not in device_map:
                logging.info(f"Il dispositivo per '{entity_id}' non esiste. Lo creo...")
                try:
                    # 1. Crea il dispositivo
                    device_payload = {'name': entity_id, 'label': f'Dispositivo HA ({entity_id})'}
                    response = session.post(f"{TB_API_URL}/api/device", json=device_payload)
                    response.raise_for_status()
                    device_id = response.json()['id']['id']
                    
                    # 2. Ottieni le credenziali (access token)
                    cred_response = session.get(f"{TB_API_URL}/api/device/{device_id}/credentials")
                    cred_response.raise_for_status()
                    access_token = cred_response.json()['credentialsId']
                    
                    device_map[entity_id] = {
                        'device_id': device_id,
                        'access_token': access_token
                    }
                    updated = True
                    logging.info(f"Dispositivo per '{entity_id}' creato con successo.")
                    
                except requests.exceptions.RequestException as e:
                    error_message = e.response.text if e.response else str(e)
                    logging.error(f"Errore nella creazione del dispositivo per '{entity_id}': {error_message}")


    if updated:
        save_json_file(DEVICES_FILE, device_map)
    logging.info("Provisioning completato.")
    return device_map




def format_ha_state(state):
    """Converte lo stato di HA in un formato adatto per ThingsBoard."""
    if state is None:
        return None
    if state.lower() in ['on', 'true', 'locked']:
        return True
    if state.lower() in ['off', 'false', 'unlocked']:
        return False
    try:
        return float(state)
    except (ValueError, TypeError):
        return state


async def ha_listener(device_map, allowed_entities):
    """Ascolta gli eventi di Home Assistant e li inoltra a ThingsBoard."""
    auth_ok = False
    subscribe_sent = False
    msg_id = 1


    uri = HA_URL
    async for websocket in websockets.connect(uri):
        logging.info("Connesso al WebSocket di Home Assistant.")
        try:
            async for message in websocket:
                msg = json.loads(message)


                if msg['type'] == 'auth_required':
                    logging.info("Autenticazione richiesta da HA...")
                    await websocket.send(json.dumps({
                        'type': 'auth',
                        'access_token': HA_TOKEN
                    }))
                
                elif msg['type'] == 'auth_ok':
                    logging.info("Autenticazione a HA riuscita.")
                    auth_ok = True
                    # Sottoscrivi agli eventi di cambio stato
                    await websocket.send(json.dumps({
                        'id': msg_id,
                        'type': 'subscribe_events',
                        'event_type': 'state_changed'
                    }))
                    subscribe_sent = True
                    msg_id += 1


                elif msg['type'] == 'auth_invalid':
                    logging.error(f"Autenticazione a HA fallita: {msg['message']}")
                    return # Termina in caso di errore di auth


                elif subscribe_sent and msg['type'] == 'event':
                    event_data = msg['event']['data']
                    entity_id = event_data['entity_id']
                    
                    if entity_id in allowed_entities:
                        new_state = format_ha_state(event_data.get('new_state', {}).get('state'))
                        if new_state is None:
                            continue
                            
                        device_info = device_map.get(entity_id)
                        if not device_info:
                            logging.warning(f"Ricevuto stato per '{entity_id}', ma non ci sono credenziali salvate.")
                            continue


                        access_token = device_info['access_token']
                        
                        try:
                            # 1. Invia sempre la telemetria principale (state)
                            telemetry_payload = json.dumps({'state': new_state})
                            publish.single(
                                topic="v1/devices/me/telemetry",
                                payload=telemetry_payload,
                                hostname=TB_MQTT_HOST,
                                port=TB_MQTT_PORT,
                                auth={'username': access_token, 'password': ''}
                            )
                            logging.info(f"Inviata telemetria per '{entity_id}': {telemetry_payload}")


                            # 2. Controlla il tipo di entità prima di inviare gli attributi
                            entity_domain = entity_id.split('.')[0]
                            if entity_domain in ['light', 'switch', 'binary_sensor']:
                                # Invia l'attributo 'running'
                                attribute_payload_running = json.dumps({'running': new_state})
                                publish.single(
                                    topic="v1/devices/me/attributes",
                                    payload=attribute_payload_running,
                                    hostname=TB_MQTT_HOST,
                                    port=TB_MQTT_PORT,
                                    auth={'username': access_token, 'password': ''}
                                )
                                
                                # Invia l'attributo 'open'
                                attribute_payload_open = json.dumps({'open': new_state})
                                publish.single(
                                    topic="v1/devices/me/attributes",
                                    payload=attribute_payload_open,
                                    hostname=TB_MQTT_HOST,
                                    port=TB_MQTT_PORT,
                                    auth={'username': access_token, 'password': ''}
                                )
                                logging.info(f"Inviati attributi 'running' e 'open' per '{entity_id}': {new_state}")


                        except Exception as e:
                            logging.error(f"Errore durante l'invio MQTT a ThingsBoard per '{entity_id}': {e}")
                            
        except websockets.ConnectionClosed:
            logging.warning("Connessione a HA persa. Tento di riconnettermi tra 5 secondi...")
            await asyncio.sleep(5)
            continue






def get_ha_rest_url():
    """Deriva l'URL REST di Home Assistant dalla HA_URL websocket se necessario."""
    url = HA_URL
    if url.startswith('ws://'):
        url = 'http://' + url[len('ws://'):]
    elif url.startswith('wss://'):
        url = 'https://' + url[len('wss://'):]
    # rimuovi il path /api/websocket se presente
    if url.endswith('/api/websocket'):
        url = url[:-len('/api/websocket')]
    # rimuovi eventuale trailing slash
    return url.rstrip('/')




def ha_call_service(entity_id, state_bool):
    """Chiama il servizio di Home Assistant per attivare/disattivare l'entità.


    Usa il service domain.turn_on/turn_off quando possibile, altrimenti imposta lo stato direttamente.
    """
    base = get_ha_rest_url()
    headers = {'Authorization': f'Bearer {HA_TOKEN}', 'Content-Type': 'application/json'}
    domain = entity_id.split('.')[0]


    # Domain-specific service mapping
    try:
        if domain == 'cover':
            service = 'open_cover' if state_bool else 'close_cover'
            service_url = f"{base}/api/services/{domain}/{service}"
            resp = requests.post(service_url, headers=headers, json={"entity_id": entity_id}, timeout=10)
            if resp.ok:
                logging.info(f"Chiamato servizio {domain}.{service} per {entity_id} -> {state_bool}")
                return True
            else:
                logging.warning(f"Servizio {domain}.{service} per {entity_id} ha restituito {resp.status_code}: {resp.text}")


        elif domain == 'climate':
            # Try to set HVAC mode as a best-effort: True -> 'heat', False -> 'off'
            hvac_mode = 'heat' if state_bool else 'off'
            service_url = f"{base}/api/services/{domain}/set_hvac_mode"
            resp = requests.post(service_url, headers=headers, json={"entity_id": entity_id, "hvac_mode": hvac_mode}, timeout=10)
            if resp.ok:
                logging.info(f"Chiamato climate.set_hvac_mode per {entity_id} -> {hvac_mode}")
                return True
            else:
                logging.warning(f"Servizio climate.set_hvac_mode per {entity_id} ha restituito {resp.status_code}: {resp.text}")


        else:
            service = 'turn_on' if state_bool else 'turn_off'
            service_url = f"{base}/api/services/{domain}/{service}"
            resp = requests.post(service_url, headers=headers, json={"entity_id": entity_id}, timeout=10)
            if resp.ok:
                logging.info(f"Chiamato servizio {domain}.{service} per {entity_id} -> {state_bool}")
                return True
            else:
                logging.warning(f"Servizio {domain}.{service} per {entity_id} ha restituito {resp.status_code}: {resp.text}")


    except requests.exceptions.RequestException as e:
        logging.error(f"Errore chiamando servizio HA per {entity_id}: {e}")


    # fallback: imposta lo stato direttamente
    try:
        state_val = 'on' if state_bool else 'off'
        resp2 = requests.post(f"{base}/api/states/{entity_id}", headers=headers, json={"state": state_val}, timeout=10)
        if resp2.status_code >= 200 and resp2.status_code < 300:
            logging.info(f"Impostato stato diretto {entity_id} -> {state_val}")
            return True
        else:
            logging.error(f"Impostazione stato diretto per {entity_id} fallita: {resp2.status_code} {resp2.text}")
    except requests.exceptions.RequestException as e:
        logging.error(f"Errore impostando stato diretto HA per {entity_id}: {e}")


    return False




def _parse_rpc_payload(payload):
    """Prova a estrarre il parametro utile dal payload RPC di ThingsBoard."""
    try:
        data = json.loads(payload)
    except Exception:
        return payload


    # Common TB RPC shapes: {"method":"set","params":true} o {"params": true}
    if isinstance(data, dict):
        if 'params' in data:
            return data['params']
        if 'value' in data:
            return data['value']
        # se è un dict con una singola chiave, prova a restituirne il valore
        if len(data) == 1:
            return list(data.values())[0]
    return data




def on_mqtt_message(client, userdata, msg):
    """Callback MQTT per RPC da ThingsBoard. user data contiene l'entity_id associata al client."""    
    try:
        payload = msg.payload.decode('utf-8')
    except Exception:
        payload = str(msg.payload)
    logging.info(f"Ricevuto RPC MQTT: topic={msg.topic} payload={payload}")


    # Parse and extract target entity or telemetry/attributes payload
    # TB can send different payload shapes:
    # - RPC: {"method":"set","params": true} or {"params": true}
    # - Attributes push: {"key": value} or via RPC-like wrapper
    # - Telemetry push (from server-side) often: {"ts":123456789,"values": {"running": true}}
    param = None
    try:
        parsed = json.loads(payload)
    except Exception:
        parsed = None


    # If telemetry envelope (ts + values)
    if isinstance(parsed, dict) and 'values' in parsed and isinstance(parsed['values'], dict):
        param = parsed['values']
    elif isinstance(parsed, dict) and ('params' in parsed or 'value' in parsed):
        # RPC-style
        param = _parse_rpc_payload(payload)
    elif isinstance(parsed, dict):
        # attribute push or simple dict
        param = parsed
    else:
        # fallback to RPC parser (handles simple true/false payloads)
        param = _parse_rpc_payload(payload)
    # if this client was created per-device, userdata will hold entity_id
    target_entity = None
    if isinstance(userdata, str) and userdata:
        target_entity = userdata
    if isinstance(param, dict):
        # common keys: entity_id, device, target
        for k in ('entity_id', 'device', 'target'):
            if k in param:
                target_entity = param[k]
                break
        # if params itself contains a value boolean under 'value' or 'params'
        if target_entity is None and 'value' in param:
            # leave target_entity None
            pass
    elif isinstance(param, str):
        # payload might be just the entity_id (rare)
        if '.' in param:
            target_entity = param


    if not target_entity:
        # try to extract from payload
        if isinstance(param, dict):
            for k in ('entity_id', 'device', 'target'):
                if k in param:
                    target_entity = param[k]
                    break
        elif isinstance(param, str) and '.' in param:
            target_entity = param


    if not target_entity:
        logging.warning(f"MQTT message ricevuto senza field 'entity_id' e senza userdata; non posso instradare: {payload}")
        return
    # determine desired boolean/value
    # param may be dict params, or an attributes dict (running/open/state)
    val = None
    if isinstance(param, dict):
        # If attribute/telemetry update contains keys like 'running'/'open'/'state', prefer them
        for key in ('running', 'open', 'state'):
            if key in param:
                val = param[key]
                break
        # otherwise fallback to params/value/command
        if val is None:
            if 'params' in param:
                val = param['params']
            elif 'value' in param:
                val = param['value']
            elif 'command' in param:
                val = param['command']
    else:
        val = param


    desired = None
    if isinstance(val, bool):
        desired = val
    elif isinstance(val, (int, float)):
        desired = bool(val)
    elif isinstance(val, str):
        low = val.strip().lower()
        if low in ['true', 'on', '1', 'yes']:
            desired = True
        elif low in ['false', 'off', '0', 'no']:
            desired = False


    if desired is None:
        logging.warning(f"Messaggio MQTT per {target_entity} con payload non interpretabile come booleano: {param}")
        return


    # call HA
    success = ha_call_service(target_entity, desired)
    if success:
        logging.info(f"Applicato stato da ThingsBoard a HA: {target_entity} -> {desired}")
    else:
        logging.error(f"Impossibile applicare stato da ThingsBoard a HA per {target_entity}")




def setup_per_device_mqtt_listeners(device_map):
    """Crea un client MQTT per ogni device (usando il suo access token) e sottoscrive RPC/attributes.
    Questo permette di ricevere attribute updates destinati esattamente a quel device, usando userdata per mappare entity_id.
    """
    clients = []
    for entity_id, info in device_map.items():
        access_token = info.get('access_token')
        if not access_token:
            logging.warning(f"Nessun access_token per {entity_id}, salto la creazione del client MQTT per questo device")
            continue


        client_id_safe = f"ha_bridge_{entity_id.replace('/', '_').replace('.', '_')}"
        client = mqtt.Client(client_id=client_id_safe)
        client.username_pw_set(username=access_token, password='')
        # set userdata so handler can know the device/entity target
        client.user_data_set(entity_id)
        client.on_message = on_mqtt_message


        def make_on_connect(eid):
            def on_connect(c, userdata, flags, rc):
                if rc == 0:
                    logging.info(f"MQTT client per {eid} connesso. Sottoscrivo RPC e attributes.")
                    c.subscribe('v1/devices/me/rpc/request/+')
                    c.subscribe('v1/devices/me/attributes')
                    c.subscribe('v1/devices/me/telemetry')
                else:
                    logging.error(f"MQTT client per {eid} fallito la connessione rc={rc}")
            return on_connect


        client.on_connect = make_on_connect(entity_id)
        try:
            client.connect(TB_MQTT_HOST, TB_MQTT_PORT, 60)
            client.loop_start()
            clients.append(client)
        except Exception as e:
            logging.error(f"Impossibile connettere MQTT client per {entity_id}: {e}")


    return clients




async def main():
    """Funzione principale per avviare il bridge."""
    if 'INCOLLA_QUI' in HA_TOKEN or not all([TB_USER, TB_PASSWORD]):
        logging.error("Configurazione mancante! Modifica i valori di default nello script (come HA_TOKEN).")
        return


    allowed_entities = load_json_file(ENTITIES_FILE, [])
    if not allowed_entities:
        logging.error(f"Il file '{ENTITIES_FILE}' è vuoto o non trovato. Nessuna entità da monitorare.")
        return
        
    device_map = load_json_file(DEVICES_FILE)
    device_map = provision_devices(allowed_entities, device_map)
    
    # Use per-device MQTT clients exclusively
    clients = setup_per_device_mqtt_listeners(device_map)
    try:
        await ha_listener(device_map, allowed_entities)
    finally:
        for client in clients:
            client.loop_stop()  # Stop MQTT clients






if __name__ == "__main__":
    asyncio.run(main())
